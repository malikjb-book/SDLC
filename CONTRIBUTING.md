# Contributing

Thank you for your interest in contributing to **AI-Augmented SDLC**!

## How to Contribute

### Suggesting Edits
1. Fork this repository
2. Create a feature branch (`git checkout -b chapter/your-topic`)
3. Make your changes
4. Submit a pull request with a clear description

### Reporting Issues
- Use GitHub Issues for errata, factual corrections, or content suggestions
- Tag issues with appropriate labels (`errata`, `enhancement`, `question`)

## Style Guide

### Writing
- Use clear, practitioner-focused language
- Prefer concrete examples over abstract theory
- Include code snippets and prompts where applicable
- Use active voice

### Formatting
- All content in Markdown (`.md`)
- One sentence per line (for better diffs)
- Use ATX-style headers (`#`, `##`, `###`)
- Place figures in the chapter's `figures/` directory
- Reference figures with relative paths

### Code Samples
- Must be self-contained and runnable
- Include comments explaining key steps
- Specify language/runtime requirements
- Place in the corresponding `code-samples/chNN-*/` directory

## Review Process

1. All contributions are reviewed by the author
2. Feedback will be provided via PR comments
3. Once approved, changes are merged into `main`

## Code of Conduct

Be respectful, constructive, and inclusive. We're all here to learn and improve.
